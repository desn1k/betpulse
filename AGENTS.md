# Repository instructions

## Communication and implementation language

- Communicate with the user in Russian.
- Discuss development plans, trade-offs, progress updates, and blockers in Russian.
- Write implementation prompts in English.
- After completing development, report in Russian what was changed and how it was validated.

## Pull request completion checklist

Before declaring a pull request ready to merge or the related work complete:

- Read all top-level PR conversation comments.
- Read all inline review comments.
- Check review threads and identify every unresolved thread.
- Re-check for comments added after the latest commit.
- Address each actionable comment, or explicitly report why it remains unresolved.
- Verify the required CI and security checks.

A green CI result alone is not sufficient: PR comments and unresolved review threads
must also be checked.
